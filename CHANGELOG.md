# Changelog

Заметные изменения проекта (инфраструктура/оркестрация). Формат: дата — суть.

## 2026-06-21 — Оркестрация: VibeProxy → Kiro CLI MCP (⚠️ в режиме теста)

### Делегирование агентам
- Мёртвый **VibeProxy** полностью убран из всех доков (неофициальный мост, привёл к бану аккаунта
  Kiro). Лог тестов `DEBUG-LOG-MULTITASK.md` удалён.
- Новый канал — официальный **Kiro CLI MCP** (`delegate`/`reply`, resume по `session_id`): первый
  приоритет в корневом `CLAUDE.md`, Codex — второй. Зафиксировано правило переиспользования сессий
  (`reply`, не плодить новые) и максимальное мышление по умолчанию.
- `ORCHESTRATION-RETROSPECTIVE.md` переоформлен из VibeProxy-ретро в живой монитор-лог под текущий
  стек (kiro-cli + Codex + Claude-Agent).

### Обёртка `~/www/kiro-cli-mcp` (hardening после ревью mine+Codex)
- Прокси **ENFORCED** (без прокси — отказ, чтобы не уйти с «голого» IP); любой `exit≠0` = ошибка
  (раньше маскировался под успех); `session_id` через delta-снимок до/после (снята гонка и выбор
  старой сессии); таймаут сохраняет частичный вывод + `session_id` для resume; UTF-8 в subprocess;
  argv-гард; redact задачи в логах.
- Пофикшен **ложный auth-баннер** «не аутентифицирован» (детект auth теперь по коду возврата —
  подстрока `kiro-cli login` в контенте правки больше не триггерит ложный сбой).

### ⚠️ Статус: В РЕЖИМЕ ТЕСТА
Новый механизм делегирования и hardening обёртки **не прошли полную живую проверку** — только
`py_compile` + смоук-тесты чистой логики; живая верификация — после реконнекта MCP. **Любые
сбои/нюансы агентов (особенно Kiro CLI) логировать в `ORCHESTRATION-RETROSPECTIVE.md`** — за этот
заход так уже выловлено несколько багов.
