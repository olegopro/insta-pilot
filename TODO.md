## Рефакторинг — архитектурные улучшения

### 1. HTTP 401 используется для двух разных ситуаций

**Проблема:**
`AuthController::login()` возвращает `401` при неверном пароле. Sanctum тоже возвращает `401` при протухшем токене. Один код — две разные причины.

**Следствие:**
На фронте в `axios.ts` нужен костыль — проверка `!isAuthLogin`, чтобы отличить «неверный пароль» от «токен невалиден».

**Решение:**
Заменить `401` на `422` в `AuthController::login()` при неверных credentials. Тогда `401` всегда означает только одно — невалидный токен, и костыль на фронте не нужен.

## Пробелы в знаниях — изучить дополнительно

- `OFFSET` в SQL — не пропускает записи, а читает и отбрасывает. На больших таблицах дорого. Альтернатива — курсорная пагинация по `id`.
- `intdiv()` в PHP — целочисленное деление без остатка.
- `??` (null coalescing) в PHP — если левая часть `null`, берётся правая.
- `use ($var)` в анонимных функциях PHP — явный захват переменных из внешней области видимости (в JS это автоматически).
- `DB::transaction(callback)` — два запроса внутри либо оба выполняются, либо ни один (`rollBack` при ошибке).
- `new Model([...])` в конструкторе — создаёт объект в памяти без сохранения в БД. `save()` потом решает INSERT или UPDATE по наличию `id`.
- `try/catch` ловит только синхронные ошибки и `await Promise.reject()`. Ошибки внутри callback (`setTimeout`, `addEventListener`, `echo.listen`) не ловятся — callback вызывается позже, когда `try/catch` уже закрыт.
- `echo.listen()` — аналог `addEventListener`, триггер не клик, а WebSocket-сообщение от бэкенда. Разобрать подробно на Backend Level 8.
- `window.scrollY` — сколько пикселей прокручено от верха. `scrollHeight` — полная высота страницы включая невидимую часть.
- `ReturnType<typeof setTimeout>` — берёт тип возвращаемого значения функции без привязки к окружению (браузер vs Node.js).
- `void` перед Promise — явно игнорирует возвращаемый Promise чтобы не было предупреждений TypeScript.
