> и обособленным от фундамента.

## 4A — Реклама / tracked (локально, безопасно)

Объём:
- Опирается на overlay-поля Phase 2: `is_ad`, `is_tracked`, `labels`. Доп. таблица `showcase_ad_records`
  (кампания/бюджет/ссылка/период) — **только если** владельцу нужны атрибуты сверх флага; для MVP хватает overlay.
- Vue: `features/mark-showcase-ad` — переключатель «реклама» на посте; бейдж «AD» в сетке; фильтр «только реклама».
- Laravel: реюз `ShowcaseOverlayController::updateOverlay` (флаги). При расширенных атрибутах — новый контроллер/таблица.

Механика:
- Пометка — чистый локальный апдейт overlay, IG не дёргается, в ActivityLog не пишется (или отдельный не-IG audit).
- `is_tracked` — задел под аналитику (Phase 5): отслеживаемые посты собирают снимки метрик.

Чек-лист 4A:
- [ ] Переключатель + бейдж + фильтр; (опц.) таблица атрибутов рекламы.
- [ ] Тесты overlay-флагов; live: пометил → бейдж/фильтр работают, переживают reload.

## 4B — Автопостинг (ОПЦИОНАЛЬНО, риск-зона — отдельное согласие)

> Не входит в «редактор профиля». Включать только по явному запросу владельца и осторожно (anti-ban).

Возможности instagrapi: `photo_upload`, `video_upload`, `album_upload`, `clip_upload`, `igtv_upload`
(+ `*_upload_to_story`). Реально публикует.

Если делать:
- Python: `POST /media/upload/photo|video|album` (приём файла/URL, caption, usertags, location).
- Laravel: загрузка медиа (валидация типов/размера), `InstagramClientService::uploadPhoto/...`, ActivityLog `publish_media`.
- Vue: форма загрузки + предпросмотр в «рамке телефона» (как встанет в сетку).
- Планирование публикаций (отложенный постинг) — переиспользовать движок очереди/диспетчер автоматизации
  (`run_at` + scheduler), НЕ изобретать заново. Отдельная под-фаза.
- Rate-limit / anti-ban: строгие лимиты, без массовости, прогрев — как в правилах безопасности проекта.

Чек-лист 4B (если активируется):
- [ ] Согласие владельца + лимиты публикаций.
- [ ] Upload-endpoint + загрузка/валидация медиа + ActivityLog.
- [ ] Live: одиночная реальная публикация на тестовом аккаунте, видна в профиле, корректно в сетке.

## Критерий готовности

4A: пометки рекламы/tracked работают локально и переживают reload. 4B (если делали): одиночная публикация
доказана на живом тестовом аккаунте под лимитами.
