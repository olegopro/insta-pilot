# CODE STYLE — python-service (FastAPI + instagrapi)

> ВНИМАНИЕ: автоформаттеров/линтеров в слое НЕТ (ни ruff/black/flake8/isort). Стиль ниже —
> зафиксированный де-факто из кода + PEP8. В legacy-файлах есть отклонения (очень длинные строки,
> смешанный typing) — это НЕ образец. Новый код пишется по правилам ниже; при правке файла —
> следовать стилю, уже принятому в ЭТОМ файле, где он не противоречит явным правилам.

## Форматирование (PEP8)
- **Отступ — 4 пробела**, никогда tab. EOL `lf`, charset `utf-8`.
- **Кавычки — двойные** `"..."` (как Black). Не смешивать в одном файле без причины.
- **Длина строки — целевая ≤120.** Длинные выражения переносить. (В legacy встречаются строки до ~224 —
  не растягивать новый код под них.)
- Docstrings — тройные двойные кавычки `"""..."""` на модулях и публичных функциях/моделях.

## Типы и импорты
- **Аннотации типов обязательны** для сигнатур функций и module-level переменных
  (`def error_to_code(exc: Exception) -> str:`, `_cache: dict[str, float] = {}`).
- **Встроенные дженерики и `X | None`** в новом коде: `dict[str, int]`, `list[str]`, `str | None` (PEP 604) —
  не плодить новые `Optional`/`List`/`Dict` из `typing` (в legacy смешано — при правке держать стиль файла).
- Импорты — **группами PEP8** (stdlib → third-party → local), группы разделены пустой строкой; в скобочном
  многострочном импорте имена по алфавиту.

## Именование
- Функции/переменные — **snake_case**; приватные/внутренние — префикс `_` (`_make_client`, `_client_cache`).
- Константы уровня модуля — **UPPER_CASE** с `_` (`_CLIENT_CACHE_TTL = 300`).
- Классы — **PascalCase**; Pydantic-модели — с суффиксами **`Request`/`Response`** (`FeedResponse`).

## Доменные конвенции (из python-service/CLAUDE.md)
- Суффиксы ID: **`_pk`** — числовой Instagram-ID, **`_id`** — составной/строковый. Не путать.
- **Не логировать** пароль, cookie, full session dump, API-ключи.
